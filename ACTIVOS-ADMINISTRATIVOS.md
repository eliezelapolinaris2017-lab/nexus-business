# Activos Administrativos

Este cambio convierte Recursos en Activos Administrativos.

## Qué cambió

- El módulo ahora se llama **Activos**.
- Cada activo puede vincularse a un cliente.
- El serial técnico no es obligatorio.
- El activo se puede relacionar opcionalmente con un servicio.
- Se añadieron reportes: **Activos por Cliente** y **Activos por Estado**.

## Campos principales

- Cliente asignado
- Nombre del activo
- Categoría
- Ubicación
- Estado
- Valor estimado
- Fecha de registro
- Garantía / vigencia
- Notas administrativas

## Flujo recomendado

1. Crear o seleccionar cliente.
2. Registrar activo administrativo.
3. Al crear un servicio, seleccionar el activo relacionado si aplica.
4. Generar reportes desde Reportes.

Esto mantiene Nexus como plataforma administrativa, no como sistema técnico especializado.
